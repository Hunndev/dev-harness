# 설계의도 작성

요구사항과 유사 구현 조사를 바탕으로 설계의도 문서를 작성한다.

## 실행 방식

이 skill은 Fork에서 실행된다.

## 절차

1. `requirements.md`와 `prior-art.md`를 읽는다.
2. `docs/adr.yaml`에서 관련 기존 결정을 확인한다.
3. 설계의도 초안을 작성한다.
4. **모호한 논의점**을 사용자에게 제시한다:
   - "A 방식과 B 방식 중 어느 쪽 의도인지?"
   - "이 예외 케이스를 의도적으로 제외한 건지?"
5. 사용자 피드백을 반영하여 확정한다.

## 산출물: design-intent.md

```markdown
# 설계의도

## 작업 개요
(이 작업이 무엇을 해결하는가)

## 핵심 설계 결정

### 결정 1: {제목}
- 선택지: A vs B
- 선택: A
- 이유: ...
- 관련 ADR: (있으면)
- 관련 prior-art: (재사용하는 기존 패턴이 있으면)

## 의도적으로 제외한 것
(이번 범위에서 빠진 것과 이유)

## 주의사항
(구현/리뷰 시 특히 신경 써야 할 점)
```

## 새 ADR 필요 판단

설계 결정 중 다음에 해당하면 **planning 트랙으로 에스컬레이션**을 제안한다:
- 기존 ADR에 없는 새로운 아키텍처 패턴 도입
- 기존 ADR과 충돌하는 결정
- 프로젝트 전체에 영향을 미치는 결정

사용자에게 에스컬레이션 여부를 확인한다. 에스컬레이션하면 파이프라인을 일시 중단한다.
