# 요구사항 인터뷰 정리

구조화된 질문을 통해 요구사항을 수집하고 분류한다.

## 실행 방식

이 skill은 Fork에서 실행된다.

## 질문 프레임

사용자에게 다음 질문을 던진다. 이미 답변된 항목은 건너뛴다.

### 문제 정의
- "이 기능이 없으면 누가 어떤 문제를 겪는가?"
- "현재 이 문제를 어떻게 우회하고 있는가?"

### 해결 형태
- "가장 단순한 형태로 만든다면 어떤 모습인가?"
- "이상적인 형태는? 단순한 형태와 어떤 차이?"

### 우선순위
- "반드시 있어야 하는 것(MUST)은?"
- "있으면 좋은 것(NICE)은?"
- "없어도 되는 것(OUT)은?"

### 연결
- "기존 시스템(예약, 결제, 커뮤니티)과 어떻게 연결되어야 하는가?"
- "기존 사용 흐름에서 어디에 끼어드는가?"

### 규모/성능
- "예상 사용자 수와 동시 접속은?"
- "데이터 양은? 시간이 지나면 어떻게 늘어나는가?"

### 제약
- "출시 시점이나 일정 제약이 있는가?"
- "예산 제약이 있는가? (Azure 비용, 외부 API 비용)"
- "법적/규제 제약이 있는가? (개인정보보호법 등)"
